# Pi Coding Agent Instructions

## 工作方式

- 默认在当前分支直接开发，除非任务明确要求新分支。
- 先读代码和配置，再动手修改；优先复用仓库已有模式。
- 搜索文件和内容优先用 `rg` / `rg --files`。
- 不要回滚用户已有改动；遇到相关改动时基于现状继续。
- 修改配置或代码后，尽量运行最小必要校验。

## Git

- 只有明确要求提交时才 commit。
- Git log 用中文，包含需求或问题、修复或实现思路；有必要时补充复现路径。

## 沟通

- 中文回复，言简意赅，可少量使用 emoji。
- 专有名词保持 English。
- 结论优先，必要时再给依据和下一步。
